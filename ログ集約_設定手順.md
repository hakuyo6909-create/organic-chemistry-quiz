# 生徒ログを先生のスプレッドシートに集約する設定手順

生徒が各自の端末で使った操作ログ（どの問題を解いた・正誤・利用時間など）を、
先生の Google スプレッドシートに**自動で集約**するための設定です。

- 費用: **無料**（Google アカウントのみ。クレカ不要）
- 送信されるもの: **出席番号と操作ログのみ**（氏名は含まれません）
- データの保存先: **先生自身の Google ドライブ**（外部業者に預けません）
- オフラインでも端末に溜めておき、ネットにつながったときに自動送信します

---

## ステップ1：集約用スプレッドシートを作る

1. Google ドライブで新しい **スプレッドシート**を1つ作成（名前は「有機化学クイズ ログ」等、任意）。
2. メニュー **拡張機能 → Apps Script** を開く。

## ステップ2：受信スクリプトを貼り付ける

1. 開いた Apps Script エディタの中身をすべて消す。
2. リポジトリの [`apps_script/Code.gs`](apps_script/Code.gs) の内容を**全部コピーして貼り付け**、保存（💾）。
3. （任意・推奨）1行目付近の `var SECRET = '';` を、合言葉に変更
   例：`var SECRET = 'kagaku2026';`

## ステップ3：ウェブアプリとしてデプロイ

1. 右上の **デプロイ → 新しいデプロイ**。
2. 歯車⚙️ → 種類 **「ウェブアプリ」** を選択。
3. 設定：
   - 次のユーザーとして実行：**自分**
   - アクセスできるユーザー：**全員**
4. **デプロイ** → 初回は Google の**承認（アクセスを許可）**を求められるので許可。
   （「このアプリは確認されていません」→ 詳細 → 安全でないページに移動、で進めます。自分のスクリプトなので問題ありません）
5. 表示された **ウェブアプリの URL**（`https://script.google.com/macros/s/……/exec`）をコピー。

> 動作確認：その URL をブラウザで開いて `OrgQuizLog receiver OK...` と出れば成功です。

## ステップ4：アプリに URL を差し込む

1. [`sync_config.js`](sync_config.js) を開く。
2. `url: ""` の `""` の中に、ステップ3でコピーした URL を貼り付け。
3. SECRET を設定した場合は `secret: ""` にも同じ合言葉を入れる。

```js
window.ORGQUIZ_SYNC = {
  url: "https://script.google.com/macros/s/AKfy...../exec",
  secret: "kagaku2026",
};
```

4. 保存して配布：
   - Web 版：`git push`（GitHub Pages が自動更新）
   - USB 版：`sync_config.js` を上書きコピー

## ステップ5：動作確認

1. アプリを開き、生徒 ID（例 `3101`）でログインしてクイズを1問解く。
2. 数十秒後、スプレッドシートの **events / sessions シート**に行が増えていれば成功。

---

## 仕組み・仕様メモ

- ログは各端末の `localStorage` に一時保存され、約20秒ごと＋ページ離脱時に送信されます。
- 送信失敗・オフライン時は溜めておき、次回オンライン時に自動再送します。
- 各行に `uid` が付き、GAS 側で6時間の重複排除を行うため、二重送信されても重複行になりません。
- `sync_config.js` の `url` が**空**なら送信機能は**完全に無効**（従来どおり端末内のみ）です。
- スプレッドシートのシート:
  - **events**：`t, studentId, sessionId, kind, mode, qid, label, given, correct, ms, extra, uid`
  - **sessions**：`studentId, sessionId, startedAt, endedAt, durationMs, autoEnded, uid`

## うまくいかないとき

- スプレッドシートに何も来ない → URL の貼り間違い、または SECRET 不一致を確認。
- URL をブラウザで開いて `OrgQuizLog receiver OK` が出るか確認。
- Apps Script を**修正したら必ず再デプロイ**（デプロイ → デプロイを管理 → 編集 → 新バージョン）。
- 生徒端末がオフラインのままだと届きません（ネット接続時にまとめて届きます）。
